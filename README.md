# Ипотечный калькулятор (SPA, HTML/CSS/JS)

Одностраничное веб-приложение для расчёта ипотеки с помесячной симуляцией и сравнением стратегий досрочного погашения.

## Возможности

- Полностью русскоязычный интерфейс.
- Расчёт аннуитетного платежа.
- Сравнение 4 сценариев:
  1. Без досрочного погашения
  2. Досрочное погашение — уменьшение срока
  3. Досрочное погашение — уменьшение платежа
  4. Комбинированная стратегия
- График остатка долга (Chart.js).
- Амортизационная таблица по месяцам.
- Переключение светлой/тёмной темы.

---

## Как запустить локально

> Приложение — статическая страница `index.html`, сборка не нужна.

### Вариант 1 (рекомендуется): через Python HTTP Server

1. Откройте терминал в папке проекта.
2. Запустите сервер:

```bash
python3 -m http.server 4173 --bind 0.0.0.0
```

3. Откройте в браузере:

```text
http://127.0.0.1:4173/index.html
```

4. Для остановки сервера нажмите `Ctrl + C` в терминале.

---

### Вариант 2: через Node.js (`serve`)

Если у вас установлен Node.js:

```bash
npx serve .
```

После запуска откройте URL, который покажет команда (обычно `http://localhost:3000`).

---

### Вариант 3: просто открыть файл

Можно открыть файл напрямую двойным кликом:

```text
index.html
```

Но предпочтительнее запуск через локальный HTTP-сервер (варианты 1–2), чтобы поведение было стабильным и одинаковым во всех браузерах.

## Как пользоваться

1. Заполните поля в блоке **«Ипотечный калькулятор»**:
   - Стоимость недвижимости
   - Первоначальный взнос
   - Ставка (% годовых)
   - Срок кредита (лет)
   - Дополнительный платёж
   - Месяц досрочного платежа
2. Выберите тип досрочного погашения.
3. Нажмите **«Рассчитать»** (или меняйте значения — расчёт обновляется автоматически).
4. Изучите:
   - карточки сравнения результатов,
   - график остатка долга,
   - амортизационную таблицу.

## Возможные проблемы и решения

### 1) Не запускается команда `python3`

Проверьте версию Python:

```bash
python3 --version
```

Если команды нет, установите Python 3 или используйте вариант с Node.js.

### 2) Порт занят

Если порт `4173` уже используется, выберите другой:

```bash
python3 -m http.server 8080 --bind 0.0.0.0
```

И откройте:

```text
http://127.0.0.1:8080/index.html
```

### 3) График не отображается

Проверьте подключение к интернету: библиотека Chart.js загружается с CDN.

## Как сделать правильный Pull Request (GitHub)

Если у вас возникли проблемы с PR (например, конфликтные маркеры `<<<<<<<`, `=======`, `>>>>>>>`), используйте этот чек-лист.

### 1) Обновите ветку и проверьте состояние

```bash
git fetch origin
git status
git branch --show-current
```

### 2) Устраните конфликты (если есть)

Проверьте файлы на конфликтные маркеры:

```bash
rg -n "^<<<<<<<|^=======|^>>>>>>>" index.html README.md
```

Если маркеры найдены:

1. Откройте файл и вручную удалите блоки с конфликтами, оставив нужный итоговый код.
2. Проверьте, что маркеров больше нет.
3. Добавьте исправленные файлы:

```bash
git add index.html README.md
```

4. Сделайте commit:

```bash
git commit -m "Resolve merge conflicts"
```

### 3) Отправьте ветку на GitHub

```bash
git push -u origin <your-branch>
```

### 4) Создайте Pull Request

#### Вариант A: через GitHub UI

- Откройте репозиторий на GitHub.
- Нажмите **Compare & pull request** для вашей ветки.
- Убедитесь, что:
  - `base` = `main` (или нужная целевая ветка),
  - `compare` = ваша feature-ветка.

#### Вариант B: через GitHub CLI

```bash
gh pr create --base main --head <your-branch> --fill
```

### 5) Финальная проверка перед отправкой

- Нет конфликтных маркеров в файлах.
- Приложение запускается локально:

```bash
python3 -m http.server 4173 --bind 0.0.0.0
```

- Страница открывается: `http://127.0.0.1:4173/index.html`.
- В PR описаны изменения и проверки.
